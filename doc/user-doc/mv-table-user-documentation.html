1 mv-table

1.1 Table definition and initialization

A table is defined using the deftable macro which has a similar syntax as defclass. An example: 

(deftable flight-path ()
  ((time :storage array
         :element-type 'float)
   (distance :storage 'foreign-vector
             :element-type 'double-float)
   (altitude :documentation "Plane altitude"
             :element-type 'number)
   (bearing :storage nil)
   (radar :element-type 'string
          :documentation "control tower identifier")
   (comment :element-type 'string))
  (:documentation "Table with a plane flight path")
  (:rows 1500)
  (:source "Some web site address")
  (:build-method :column-wise))

The columns of this table are either native vectors, foreign-arrays (from the Antic/Grid library), or undefined, meaning they can be either of the two. The element type in the columns is also defined, and is supposed to be a valid CL type.

With the table defined, we can make an instance of it, and load it 

(setf flight-xyz (make-table 'flight-path)
      (time flight-xyz) a-native-vector
      (distance flight-xyz) a-foreign-vector
      (bearing flight-xyz) yet-another-vector)

1.2 Accessing table contents

Table contents can be accessed in two ways.

  • A table-column is accessed using the predefined accessor. This returns the data structure which stores the data: a native or a foreign vector. For example
    • (distance flight-table)
  • A table-row is accessed via row-accessor object.

1.2.1 Table row accessors

A table row accessor is an object that access contents of a table row. To access a row element, we say 

(distance table-row)
(setf (comment table-row) "a comment")

Table rows objects are created like this: 

(table-row table :where selector-function)

The selector-function is a function that returns true or false for each row. It's discussed in Ch. 27 of PCL.

1.3 Table saving and loading

Tables are serializable: they can be saved into a file and loaded in a later CL session. 

(save-table stream table)
(load-table stream)

1.4 Selecting parts of a table

Note: This section is motivated by Chapter 27 of Practical Common Lisp.

From a table we can derive another table with a sub set of columns or rows. This is done with the select command. For example, here we get a new table with the time and altitude data while the flight was under JFK's airport control (with apologies to the correct nomenclature) 

(select flight-xyz :columns '(time altitude)
        :where (matching flight-xyz (= control-tower "JKF")))

1.5 Deriving and combining tables, creating summaries

A table can be derived from values in one or more tables. We can also obtain one or more summaries of a table. This is done with functions and methods whose arguments are tables.
